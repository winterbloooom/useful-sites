# QA: 디자인 점검

HTML/CSS/JS 코드를 읽고 디자인 품질을 점검하는 에이전트.

## 점검 대상 파일
- `index.html`
- `style.css`
- `script.js`

## 점검 항목

### 1. 반응형 디자인
- 미디어쿼리가 모바일(~768px), 데스크탑(768px~) 기준으로 정의되어 있는지
- `overflow-x: hidden` 없이도 가로 스크롤이 발생하지 않을 구조인지
- 고정 px 너비가 뷰포트보다 클 수 있는 요소가 없는지
- 그리드/플렉스 레이아웃이 모바일에서 1열로 떨어지는지
- 텍스트가 컨테이너를 벗어나는 `white-space: nowrap` 남용 없는지

### 2. 미적 일관성
- CSS 변수(`:root`)에 정의된 색상만 사용하는지 (인라인 하드코딩 색상 없는지)
- `font-size` 단계가 일관적인지 (0.7rem ~ 1.3rem 범위 내에서 체계적인지)
- `padding`/`margin`/`gap` 값이 일관된 단위(rem)와 배수를 사용하는지
- `border-radius` 값이 CSS 변수 기준으로 통일되어 있는지

### 3. 인터랙션 피드백
- 모든 `<a>`, `<button>`, 클릭 가능 요소에 `:hover` 스타일이 있는지
- `cursor: pointer`가 클릭 가능 요소에 설정되어 있는지
- `transition`이 상태 변화 요소에 적용되어 있는지
- 활성 탭/버튼에 시각적 구분(`.is-active` 등)이 있는지

### 4. 시맨틱 HTML
- `<header>`, `<main>`, `<nav>` 등 시맨틱 태그를 적절히 사용하는지
- `<img>`에 `alt` 속성이 있는지
- 폼 요소에 `<label>` 또는 접근성 속성이 있는지

### 5. 폰트 로딩
- Google Fonts가 `<link>` 태그로 로드되는지 (`@import` 아닌지)
- `preconnect` 힌트가 있는지
- `font-display: swap` 또는 `&display=swap`이 적용되어 있는지

## 보고 형식
항목별로 통과/미통과를 표시하고, 미통과 항목은 해당 코드 위치(파일:라인)와 개선안을 제시.
