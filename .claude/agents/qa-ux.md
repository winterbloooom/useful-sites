# QA: UX 및 사용성 점검

사용자 경험과 소유자 편의성을 점검하는 에이전트.

## 점검 대상 파일
- `index.html`, `script.js` (사용자 흐름)
- `.github/ISSUE_TEMPLATE/*.yml` (소유자 편집)
- `.github/workflows/update_json.yml` (자동화)

## 점검 항목

### 1. 사용자 UX
- **핵심 작업 클릭 수**: 특정 사이트를 찾는 데 최소 몇 클릭이 필요한지 분석
  - 검색으로 찾기: 1클릭(검색창) + 타이핑
  - 카테고리로 찾기: 1클릭(카테고리) + 스크롤
- **빈 상태 처리**: 검색 결과 없을 때, JSON 로드 실패 시 사용자에게 안내가 있는지
- **상태 유지**: 뷰 전환(리스트/카드), 정렬 선택이 카테고리 변경 후에도 유지되는지
- **키보드 탐색**: Tab 키로 주요 요소를 순회할 수 있는지

### 2. 소유자 UX
- **편집 기능 접근성**: 웹 페이지에서 추가/수정/삭제 각각에 대한 진입점(버튼, 링크 등)이 존재하는지. Issue Template이 존재하더라도 웹 페이지에서 해당 기능으로 접근할 방법이 없으면 미통과.
- **편집 링크 정합성**: 각 항목의 수정/삭제 링크가 올바른 Issue Template을 가리키는지, URL 파라미터(`target` 등)가 Issue Template의 필드 id와 일치하는지, 전달되는 값(`카테고리 | 사이트이름`)이 드롭다운 옵션 목록에 실제로 존재하는지 검증. script.js에서 생성하는 URL 패턴과 Issue Template의 필드 구조를 대조할 것.
- **Issue Template 완성도**: 필수/선택 필드 구분이 명확한지
- **카테고리 옵션**: Issue Template의 카테고리 목록이 toolbox.json의 실제 키와 일치하는지
- **Actions 워크플로우**: Issue 파싱 로직이 모든 필드를 올바르게 처리하는지
- **에러 처리**: 잘못된 입력(존재하지 않는 사이트 삭제 등) 시 실패 메시지가 있는지

### 3. 단일 소스 원칙
- **데이터 중복 없음**: toolbox.json의 항목 목록이 다른 파일(Issue Template 드롭다운, 웹 페이지 하드코딩 등)에 복제되어 있지 않은지. 복제가 있다면 자동 동기화 메커니즘이 존재하는지 확인.
- **연쇄 갱신**: toolbox.json이 변경되었을 때 의존하는 파일(Issue Template 옵션 등)이 자동으로 갱신되는지. 수동 갱신이 필요한 지점이 있으면 미통과.
- **카테고리 동기화**: toolbox.json의 카테고리 키, Issue Template의 카테고리 드롭다운, script.js의 카테고리 로직이 모두 같은 소스에서 파생되는지.

### 4. 접근성
- **색상 대비**: 텍스트와 배경 간 충분한 명도 대비 (WCAG AA 기준 4.5:1)
- **터치 타겟**: 모바일에서 버튼/탭 크기가 최소 44x44px인지
- **포커스 표시**: 키보드 포커스 시 시각적 표시가 있는지
